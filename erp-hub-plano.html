⚙️ Do "pronto na gaveta" ao "rodando"

São só coisas que você faz (infra é sua). Marque conforme faz — salva no seu aparelho.

Regra: tudo em contas SUAS (lepecosta1). Nunca a senha em código — as chaves vão no arquivo .env.local.

1. Supabase (banco + login)

Criar conta em supabase.com → New project (região South America / São Paulo)
Em Project Settings → API, copiar: Project URL, anon key, service_role key
Em SQL Editor, rodar o supabase/schema.sql (cria tabelas + RLS)
Depois rodar o supabase/02_auth.sql (cria empresa no 1º acesso)

2. App do Bling (integração)

Entrar em developer.bling.com.br → criar um aplicativo
Copiar client_id e client_secret
Definir a URL de redirecionamento: http://localhost:3000/api/bling/callback (em produção troca por a da Vercel)
Na conta da Folky no Bling, conectar ML e TikTok (para os pedidos entrarem no Bling)

3. Stripe (assinatura — pode deixar por último)

Conta em stripe.com → criar 1 produto "Assinatura ERP-hub" com preço mensal
Copiar Secret key e o Price ID

4. Ligar tudo (.env.local)

No C:\erp-hub-app, copiar .env.example para .env.local e colar todas as chaves
Inventar uma frase secreta longa para ENCRYPTION_KEY (cifra os tokens do cliente)

5. Rodar

cd C:\erp-hub-app npm install npm run dev
Abrir http://localhost:3000 → criar conta → criar empresa (Folky)
Testar sem Bling: botão "Gerar dados de exemplo" → margem aparece
Testar real: Conectar Bling → Importar vendas → margem e conciliação da Folky
Me chama quando o Supabase estiver de pé que eu faço o passo a passo junto e a gente vê rodando de ponta a ponta.

🚀 O iceberg — construir em ondas

Cada onda = receita nova no mesmo cliente. Só entra quando a anterior está viva e vendendo. É o que blinda contra dispersão.

Onda 0 — o MVP (agora)

Folky com dado real · margem por canal · conciliação de repasses. O degrau que vira "cliente pagando".

Onda 1 — fecha o operacional

Expedidor · NF + DANFE + etiqueta + coleta ("nenhum pedido fica pra trás")
Central de atendimento · caixa única ML+TikTok+Shopee, IA rascunha, humano envia

Onda 2 — inteligência de venda

Radar de anúncios · performance, reputação, CTR por anúncio
Precificação inteligente · sugere preço por margem-alvo (IA sugere, humano aprova)
Estoque regulador · prevê ruptura e sugere reposição

Onda 3 — o que ninguém entrega bem

BI mobile-first · "a IA narra sua semana"
CRM de marketplace · recompra, recorrência, pós-venda
Multi-loja / franquia · mesma engine, várias operações (nasce do multi-tenant)

Onda 4 — plataforma

Marketplace de conectores · Shopee, Amazon, Shein, Nuvemshop
Camada agêntica · agentes coordenando tudo, humano no loop no que é dinheiro

🧠 A crítica que eu mantenho

Não lidere pelos agentes. "Tudo automatizado com IA" é o destino, não o arranque. Agente sobre core vazio automatiza o nada. Ordem que faz faturar: core vivo → valor provado → depois a camada agêntica. Vou te lembrar disso toda vez que a tentação de pular etapa aparecer.
Princípio inegociável: IA nunca decide dinheiro sozinha. Prepara e sugere; o humano confirma o que paga, produz ou responde.
