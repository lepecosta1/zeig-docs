🛡️ Plano de segurança (backup + rollback)

Antes de QUALQUER coisa subir:

1. Backup do banco (Firestore produção) — exporto um snapshot completo. Se der ruim, restauramos.

2. Tag do código — marco o commit atual de produção (ponto de retorno).

3. Rollback do site — o Netlify guarda os deploys; reverter é 1 clique (volta pro anterior na hora).

Nosso fix muda a LÓGICA de importação, não migra dados existentes. Então o risco ao que já está no banco é baixo — o rollback de código já resolve 90% dos cenários.
O único passo que MEXE em dados (re-sync de vendas antigas) só roda DEPOIS do backup, e separado do deploy — nunca junto.
